Blog Post Categories

Categories keep posts organized and easy to navigate.

Adding a New Category
  1. Go to Posts → Categories in WP-Admin.
  2. Enter a Name (how it appears on the site) and Slug (URL-friendly version — auto-generated if left blank).
  3. Optionally set a Parent Category for subcategories.
  4. Click Add New Category.
Setting a Default Category

Go to Settings → Writing and select the Default Post Category from the dropdown. All new posts are automatically assigned this category.

Assigning a Category to a Post
  1. Open the post, then open the post settings sidebar.
  2. Scroll to Categories and check the box next to the category you want.
  3. Click Save.

Titles vs. Permalinks

When you create a page, WordPress auto-generates a permalink (slug) from the title. Once set, the permalink stays the same even if you change the title — meaning the two can differ intentionally.

A page title and permalink showing how they can differ

Example: the URL is kickpoint.ca/newsletter/ while the page title reads A Newsletter You'll Actually Read.

Important Don't change a permalink after publishing — it will break any existing links to the page and can cause a loss of search traffic. Page titles can be changed freely.

Saving Your Work

The Save Draft, Preview, and Publish buttons at the top right

Save Draft

Saves your content without publishing it. Use this to save progress or return to a page later.

Publish

Saves and makes your content live. Visibility options:

  • Public — live to all visitors (default)
  • Private — visible only to admin-level users
  • Password protected — requires a password to view

You can set a publish date in the past (backdate) or schedule for the future. After the first publish, this button changes to Update.

Published too soon? "Switch to draft" will appear where "Save Draft" was — click it to unpublish.

Preview

Opens a new tab showing what the page would look like if published. Use this to check formatting before committing.

Templates

Set page templates in the Page Summary sidebar. For most new pages, leave the template as Default.

The template selector in the page attributes sidebar

Available Templates

  • Default — Most flexible; use for the majority of pages. Automatically adds a banner block.
  • About — Same as Default, plus About page schema and an intro text block below the banner.
  • Contact — Pre-set on the Contact page. Shows contact info on the left and a contact form on the right.

Adding a Pillar Page

The Pillar template creates a sticky sidebar table of contents. To build one:

  1. Add a Section Heading block — this text appears as a link in the sidebar TOC.
  2. Add content blocks below it as needed.
  3. Repeat for each new section.
Example of a Pillar page with sticky sidebar table of contents

If you want the sidebar link text to differ from the on-page heading, use the Alternative Heading field in the Section Heading block settings.

Setting a Parent Page

Pages are built using modular content blocks in the flex builder. Every new page starts with a Banner block added automatically.

Note If you don't edit the Banner block, it will automatically use the page title as the H1 and display the Basic banner style.

Adding a Block

Click the + Block Inserter button, or hover between existing blocks to reveal another inserter. You can also type / to search by block name and press Enter to insert. Select Browse All to scroll through every available block.

The content block inserter with search bar

Moving, Removing, or Duplicating a Block

Select a block to reveal its toolbar. Use the up/down arrows to reorder, or the six-dot icon to drag and drop. Click the three-dot menu (⋯) to duplicate or delete.

The content block toolbar and dropdown menu

Reusable Blocks (Patterns)

Create a Reusable Block

  1. Select the blocks you want to reuse.
  2. Click the three-dot menu → Create pattern.
  3. Name it. Optionally assign a category.
  4. Choose Synced (edits propagate everywhere) or Unsynced (edits apply only to that instance).
  5. Click Create.

Edit a Reusable Block

Go to Appearance → Patterns, find your pattern, and click the pencil icon. Save when done — synced patterns update everywhere they're used.

Detach a Reusable Block

Select the pattern on a page → click ⋯ in the toolbar → Detach. Converts that instance to unsynced, leaving the original untouched.

The Link Pop-up

Many blocks have a "Select Link" button that opens a link pop-up. To link to a page on your site, search for it using the search box. To link externally, paste the URL directly. Always check Open link in a new tab for external links.

Adding a Table

Tables are managed through TablePress in the admin menu.

Creating a New Table

  1. Go to TablePress → Add New.
  2. Enter a name and the number of rows and columns, then click Add Table.
  3. Fill in the cells. The table is assigned an ID you'll use to insert it.
  4. Set options: header row, alternating row colours, sorting, pagination.
  5. Click Save Changes. Use Preview to check the result.
To show the table name above the table, check Print Table Name in the table options.

Importing a Table

Go to TablePress → Import a Table. Select your source (Excel, CSV, Google Sheets export, etc.) and click Import Table. You'll land on the edit page with content pre-filled.

Inserting a Table into a Page

Use the Table block and enter the table's ID, or insert the shortcode directly into a text field:

[table id=<ID> /]

Find a table's ID under TablePress → Show Shortcode.

Adding a Button

Add a button to any text editor block using this shortcode:

[kpbutton href="URL HERE" class="btn" target="_blank"]Button Text[/kpbutton]
  • href — the destination URL
  • target="_blank" — opens in a new tab; use _self to open in the same tab
  • Button Text — the text that appears on the button

Appendix: The Media Library

Images, SVGs, and PDFs are managed under Media in the left sidebar. Switch between list and grid views using the icons at the upper left.

Uploading Files

Click Add New and drag and drop files, or click Select Files to upload from your computer.

File naming Use dashes (-) not spaces or underscores. Avoid accented characters (ä, è) and special characters (%, ^, *, /).

Adding Alt Text

Alt text is used by screen readers and helps search engines understand image content. Add it when uploading via the Image block (right sidebar), or via Media → Library by clicking the image to edit it.

Appendix: Title Tags & Meta Descriptions

Page Titles

Be compelling and include your target keyword(s). Keep titles between 40–60 characters so Google doesn't truncate them. Use Title Case. Use Moz's Title Tag Preview Tool to preview how your title will appear in search results.

Meta Descriptions

Don't directly affect rankings, but a well-written description improves click-through rates. Guidelines:

  • Under 300 characters
  • Include what the page offers and a call to action
  • Unique for every page
  • Active voice; include keywords naturally

Determining Keywords

  1. Think about what someone would Google to find this specific page.
  2. Search those terms and study the top results. What are they doing well?
  3. Check "Searches related to…" at the bottom of the Google results page for additional keyword ideas.
  4. Weave keywords naturally into the title and page content — don't force awkward phrasing.
  5. Target different keywords on different pages. Don't compete against yourself.

Appendix: Adding Users

Found under Users in the left sidebar. Administrative access only.

Creating a New User

  1. Go to Users → Add New.
  2. Choose a username — this is permanent and cannot be changed later.
  3. Enter the user's email address.
  4. Select Editor from the Role dropdown (unless you have a specific reason to assign a higher role).
  5. Leave Send User Notification checked so they receive login instructions.
  6. Click Add New User.

Removing a User

  1. Hover over the user's name on the Users page → click Delete.
  2. On the Delete User screen, assign their content to another user before confirming.
Warning If you delete a user without reassigning their content, all posts and pages they created will be permanently deleted.

Appendix: Adding Links By Hand

For text fields without a formatting toolbar, write links using HTML:

Email Link

No spaces or dashes in the number. Include the country code (+1) for calls from outside your area.

Appendix: Adding Redirects

Redirects are managed under Tools → Redirection.

Redirect a Removed Page

  1. In Add New Redirection, enter the removed page's path in Source URL.
    Example: strip the domain from https://yoursite.com/case-studies/ → enter /case-studies/
  2. Enter the full destination URL in Target URL.
  3. Click Add Redirection.

Redirect for an Updated Permalink

When you change a page's slug, the Redirection plugin creates the redirect automatically.

Vanity URL Redirect

The Source URL doesn't have to be a real page — just make sure it doesn't conflict with an existing URL.

Avoid redirect chains Search for existing redirects before creating new ones. Chains (A→B→C) slow page loads and hurt SEO. If a chain exists, update the first redirect to point directly to the final destination.
